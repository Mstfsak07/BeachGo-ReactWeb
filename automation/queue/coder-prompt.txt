# FAZ 1: Authentication ve Güvenlik Temeli - Kod Geliştirme

## Amaç
Proje analizinde tespit edilen en kritik eksikler olan Authentication akışındaki güvenlik açıklarını, eksik token yönetimini ve temel auth sayfalarındaki sorunları düzelt.

## Ön Analiz - Okunacak Dosyalar

Executor önce şu dosyaları okusun ve mevcut durumu tespit etsin:

1. `C:\Users\akMuratNET\Desktop\BeachGo\BeachGo-ReactWeb\beach-ui\src\api\axios.js`
2. `C:\Users\akMuratNET\Desktop\BeachGo\BeachGo-ReactWeb\beach-ui\src\context\AuthContext.jsx`
3. `C:\Users\akMuratNET\Desktop\BeachGo\BeachGo-ReactWeb\beach-ui\src\pages\Login.jsx`
4. `C:\Users\akMuratNET\Desktop\BeachGo\BeachGo-ReactWeb\beach-ui\src\pages\Register.jsx`
5. `C:\Users\akMuratNET\Desktop\BeachGo\BeachGo-ReactWeb\beach-ui\src\App.js`
6. `C:\Users\akMuratNET\Desktop\BeachGo\BeachGo-ReactWeb\beach-ui\src\components\PrivateRoute.jsx` (varsa)
7. `C:\Users\akMuratNET\Desktop\BeachGo\BeachGo-ReactWeb\beach-ui\src\components\Navbar.jsx` (varsa)

Eğer dosyalar farklı uzantıda (.js, .tsx) ise o uzantıyla dene.

## Yapılacak Değişiklikler

### 1. `src/api/axios.js` - Düzeltme ve İyileştirme

Mevcut dosyayı oku, ardından şu sorunları düzelt:

**Eklenecekler:**
- `refreshAccessToken` fonksiyonunu düzgün şekilde `export` et (şu an export edilmiyor)
- Request interceptor: Her istekte localStorage'dan token alıp Authorization header'a ekle
- Response interceptor: 401 gelirse token refresh dene, refresh de başarısız olursa logout yap
- Token expire kontrolü (JWT decode veya basit zaman kontrolü)
- baseURL'i environment variable'dan oku: `process.env.REACT_APP_API_URL || 'http://localhost:5000/api'`

**Örnek yapı:**
```
- axiosInstance (default export)
- refreshAccessToken (named export)
- Interceptor: request → token ekle
- Interceptor: response → 401 ise refresh → retry → fail ise logout
```

### 2. `src/context/AuthContext.jsx` - Güçlendirme

Mevcut dosyayı oku, ardından şu eksikleri ekle:

**Eklenecekler:**
- `loading` state: Uygulama başında token kontrolü sırasında loading=true olsun, bittikten sonra false
- `isAuthenticated` computed değeri veya state: `user !== null` kontrolü
- `login` fonksiyonu: API'dan gelen `accessToken` ve `refreshToken`'ı localStorage'a kaydet, user state'ini set et
- `logout` fonksiyonu: localStorage'ı temizle, user state'ini null yap, `/login`'e yönlendir
- `register` fonksiyonu: API çağrısı yap, başarılıysa login yap veya doğrulama mesajı göster
- useEffect: Uygulama başında localStorage'daki token'ı kontrol et, geçerliyse user'ı set et
- Token yenileme: refresh token varsa accessToken yenile

**Korunacaklar:**
- Mevcut çalışan mantığı bozmadan ekle

### 3. `src/pages/Login.jsx` - Form İyileştirme

Mevcut dosyayı oku, ardından şu eksikleri ekle:

**Eklenecekler:**
- Form validation: email format kontrolü, password en az 6 karakter
- Loading state: `isLoading` → submit sırasında buton disabled + spinner
- Error state: API hatasını kullanıcıya göster (kırmızı alert kutusu)
- "Şifremi Unuttum" linki → `/forgot-password` route'una (sayfa sonradan yapılacak, link ekle)
- Başarılı login sonrası `navigate('/home')` veya `navigate(from, { replace: true })` ile önceki sayfaya dön
- AuthContext'ten `login` fonksiyonunu kullan

### 4. `src/pages/Register.jsx` - Form İyileştirme

Mevcut dosyayı oku, ardından şu eksikleri ekle:

**Eklenecekler:**
- Form validation: Ad/soyad zorunlu, email format, password en az 6 karakter, şifre tekrar eşleşme kontrolü
- Loading state: submit sırasında buton disabled + "Kaydediliyor..." yazısı
- Error state: API hatasını kullanıcıya göster
- Başarılı kayıt sonrası: "E-posta doğrulama gönderdik" mesajı göster
- AuthContext'ten `register` fonksiyonunu kullan

### 5. `src/components/PrivateRoute.jsx` - Eksikse Oluştur

Dosya yoksa yeni oluştur, varsa gözden geçir:

**Amaç:** Giriş yapmamış kullanıcıları `/login`'e yönlendir. Login sonrası geri dön.

**İçerik:**
- AuthContext'ten `user` ve `loading` al
- `loading` true ise: spinner veya boş div döndür
- `user` null ise: `<Navigate to="/login" state={{ from: location }} replace />`
- User varsa: `<Outlet />` döndür

### 6. `src/components/Navbar.jsx` - Auth Durumuna Göre Menü

Mevcut dosyayı oku, ardından:

**Eklenecekler:**
- Giriş yapmamışsa: "Giriş Yap" ve "Kayıt Ol" butonları göster
- Giriş yapmışsa: Kullanıcı adı/foto, "Profilim", "Rezervasyonlarım", "Favorilerim", "Çıkış" göster
- AuthContext'ten `user` ve `logout` al
- Mobil için hamburger menü (mevcut yapıya göre)

### 7. `src/App.js` - Route Yapısını Düzelt

Mevcut dosyayı oku, ardından:

**Eklenecekler:**
- PrivateRoute wrapper ile korunan route'lar:
  - `/profile` → Profile sayfası (yoksa placeholder)
  - `/reservations` → Reservations sayfası (yoksa placeholder)
  - `/favorites` → Favorites sayfası (yoksa placeholder)
- Public route'lar:
  - `/` → Home
  - `/login` → Login
  - `/register` → Register
  - `/beaches/:id` → BeachDetail
  - `/forgot-password` → ForgotPassword (placeholder sayfa)
- AuthContext'ten gelen `loading` state ile LoadingSpinner göster

## Dosya Oluşturma/Düzenleme Sırası

1. `src/api/axios.js` → Düzenle

3. `src/components/PrivateRoute.jsx` → Oluştur veya düzenle
4. `src/pages/Login.jsx` → Düzenle
5. `src/pages/Register.jsx` → Düzenle
6. `src/components/Navbar.jsx` → Düzenle
7. `src/App.js` → Düzenle

## Kritik Kurallar

- Hiçbir sayfayı silme, sadece iyileştir
- Mevcut çalışan kodları bozmadan ekle
- Yoksa placeholder sayfalar için basit `<div>Yakında</div>` yeterli
- Her dosyayı önce oku, sonra yaz
- Türkçe karakterleri doğru kullan (UTF-8)
- localStorage key isimleri: `accessToken`, `refreshToken`, `user`
- API endpoint'leri: `/auth/login`, `/auth/register`, `/auth/refresh`

## Başarı Kriterleri

- [ ] axios.js'de request/response interceptor var
- [ ] AuthContext'te loading, user, login, logout, register var
- [ ] Login formunda validation ve hata mesajı var
- [ ] Register formunda validation ve hata mesajı var
- [ ] PrivateRoute korumalı route çalışıyor
